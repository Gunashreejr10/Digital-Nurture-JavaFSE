1. Setting Up Bootstrap 5

Exercise 1.1 — Basic HTML page linked to Bootstrap via CDN

This is exactly what this page does. The two lines that matter are in the <head>:

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

The CSS goes in <head>, the JS bundle goes right before </body> (see the bottom of this file).

Exercise 1.2 — npm / downloaded Bootstrap files

Two valid ways to do this instead of the CDN:

Option A — npm (used with a bundler like Vite/webpack): 
npm init -y
npm install bootstrap @popperjs/core

Then in your JS entry file:

import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap/dist/js/bootstrap.bundle.min.js';
Option B — downloaded files (no build tool):

Download the zip from getbootstrap.com, unzip it next to your HTML file, then reference the local files instead of the CDN:

<link href="bootstrap-5.3.3-dist/css/bootstrap.min.css" rel="stylesheet">
<script src="bootstrap-5.3.3-dist/js/bootstrap.bundle.min.js"></script>

Sample folder used for this exercise: /bootstrap-5.3.3-dist/css/, /js/ — see the note in Exercise 2.1 below for what's inside.

2. Bootstrap Structure and Files

Exercise 2.1 — Directory structure explained

bootstrap-5.3.3-dist/
├── css/
│   ├── bootstrap.css            (uncompiled, readable — good for learning/debugging)
│   ├── bootstrap.min.css        (minified — use this in production)
│   ├── bootstrap-grid.css       (just the grid system, nothing else)
│   ├── bootstrap-utilities.css  (just the utility classes)
│   └── bootstrap-reboot.css     (just the CSS reset)
├── js/
│   ├── bootstrap.js              (plain JS, needs Popper loaded separately)
│   ├── bootstrap.min.js
│   ├── bootstrap.bundle.js       (JS + Popper.js bundled together)
│   └── bootstrap.bundle.min.js   (what most projects should actually use)
└── icons/                        (only present if you download Bootstrap Icons separately)
    ├── font/
    └── bootstrap-icons.svg

css/ holds the stylesheets that control layout, components, and utilities.

js/ holds the JavaScript that powers interactive components (modals, dropdowns, tooltips, accordions, carousels). Popper.js is required for anything that gets positioned dynamically, like dropdowns and tooltips — that's why the "bundle" versions exist, so you don't have to load Popper separately.

icons/ holds the icon font/SVG files if you're not pulling icons from the CDN.

19. Customization with Sass

Exercise 19.1 — Set up Bootstrap 5 with Sass via npm

mkdir bootstrap-sass-project && cd bootstrap-sass-project
npm init -y
npm install bootstrap sass --save-dev

Create src/custom.scss:

// src/custom.scss
@import "../node_modules/bootstrap/scss/bootstrap";

Add a build script to package.json:

"scripts": {
  "build:css": "sass src/custom.scss dist/custom.css"
}

Exercise 19.2 — Customize colors and border radius, then recompile

Override Bootstrap's Sass variables before importing Bootstrap itself — order matters:

// src/custom.scss

// 1. Overrides FIRST
$primary: #7b2cbf;
$border-radius: 1rem;
$border-radius-lg: 1.5rem;
$border-radius-sm: 0.5rem;

// 2. THEN import Bootstrap so it picks up the overrides
@import "../node_modules/bootstrap/scss/bootstrap";

Recompile with:

npm run build:css

Every component that references $primary (buttons, links, badges, etc.) and every rounded corner now uses the new values automatically — that's the main advantage of Sass customization over overriding CSS after the fact.
